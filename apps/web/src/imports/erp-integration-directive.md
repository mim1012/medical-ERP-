# ERP 프로그램 최종 통합 지시서

이번 프로젝트는 **쇼핑몰이 아니라 ERP 프로그램**이다.
따라서 쿠폰, 타임딜, 리뷰, 찜, 프로모션, 소비자 주문, 배너형 마케팅, 커머스 기능은 **절대 포함하지 않는다.**

이번 목표는 **영업용 시안이 아니라 실제로 운영 가능한 ERP 프로그램을 완성하는 것**이다.
기존 폴더 구조를 기반으로 하되, 지금까지 정리한 원칙들을 모두 반영하여 **개발·서버·도메인까지 실제 서비스 가능 수준으로 설계**한다.

---

## 1. 프로젝트 최종 목표

이 프로그램은 다음을 만족해야 한다.

* 실제 운영 가능한 ERP 프로그램
* 관리자와 실무 작업자가 매일 사용하는 구조
* 거래처, 품목, 재고, 출고, 세금계산서, 정산이 자연스럽게 연결되는 구조
* **품목 기준(Product-first) 출고**를 핵심 업무 흐름으로 하는 구조
* 웹 기준으로 완성하되 모바일 반응형까지 고려
* 도메인 연결 후 서버 배포 가능한 구조
* 개발자가 바로 구현 가능한 수준의 구조와 규칙을 포함
* 나중에 확장 가능하지만, 현재 버전만으로도 충분히 사용 가능한 수준

* 쿠폰

* 찜
* 소비자 주문 UI

## 5. 핵심 업무 구조

이번 ERP의 가장 중요한 구조는 아래다.

### 핵심 구조

**품목 먼저 선택 → 해당 품목에 연결된 거래처 표시 → 거래처 선택 → 판매/출고 진행 → 세금계산서 → 정산 연결**

즉 기존 일반적인 거래처-first 구조가 아니라,
이번 ERP는 **Product-first 출고 구조**를 핵심으로 한다.

이 구조를 통해 다음을 가능하게 한다.

* 어떤 품목을 내보낼지 먼저 판단
* 해당 품목과 연결된 거래처 확인
* 거래처별 기본단가 자동 반영
* 소모품/장비 구분에 따라 lot/serial 자동 분기
* 출고 완료 후 세금계산서와 정산까지 자연스럽게 연결

### 7-6. 출고/판매 관리

이번 프로젝트의 핵심 화면이다.

#### 핵심 원칙

* 거래처-first가 아니라 **품목-first**
* 품목 선택 후 연결 거래처 표시
* 거래처 선택 후 출고 입력
* 세금계산서와 정산 연결까지 고려

#### 데스크탑 화면 구조

3구역으로 구성한다.

1. 좌측: 품목 선택
2. 중앙: 연결 거래처 목록
3. 우측: 출고 입력 및 확인 패널

---

#### 좌측: 품목 선택 영역

기능:

* 품목 검색
* 품목 리스트 표시
* 품목 선택

표시 정보:

* 품목명
* 품목코드
* 품목유형
* 사용가능재고
* lot 사용 여부
* serial 사용 여부

---

#### 중앙: 연결 거래처 목록

품목 선택 시 자동 노출

표시 컬럼:

* 거래처명
* 담당자명
* 기본단가
* 최근 거래 여부
* 미수금 유무
* 상태

상태 의미:

* 정상: 출고 가능
* 주의: 미수금 있음
* 중지: 출고 불가

원칙:

* 중지는 선택 불가 처리 가능
* 주의는 경고만 표시하고 선택은 가능

---

#### 우측: 출고 입력 및 확인 패널

거래처 선택 시 자동 반영:

* 품목명
* 거래처명
* 기본단가

입력 규칙:

* 소모품: lot 선택 + 수량 입력
* 장비: serial 선택
* 필요 시 세액 자동 계산
* 공급가, 세액, 합계 자동 계산

최종 표시:

* 품목명
* 거래처명
* lot 또는 serial
* 수량
* 공급가
* 세액
* 합계
* 미수금 경고
* 세금계산서 상태

버튼:

* 임시저장
* 출고확정
* 취소

출고 상태:

* 임시저장
* 확정
* 취소

운영 규칙:

* 출고확정 시 세금계산서 상태는 발행대기
* 출고확정 데이터는 정산 연결 가능 상태가 됨

---

#### 모바일 출고 구조

모바일은 단계형으로 단순화한다.

1. 품목 선택
2. 거래처 선택
3. 출고 입력
4. 확인 및 확정

원칙:

* 복잡한 3패널 축소 금지
* 모바일은 단계형 흐름으로 단순화
* 반응형 웹으로 구현 가능하도록 구성

## 9. 개발 구조 원칙

이번 프로그램은 실제 서버 연동과 도메인 연결을 고려하므로, 프론트 구조는 단순 시안이 아니라 개발 가능한 구조로 간다.

### 프론트 원칙

* 페이지 단위 분리
* 공통 컴포넌트 분리
* 상태값 명확화
* 메뉴별 CRUD 구조 명확화
* 테이블 / 폼 / 상세패널 공통화
* 모바일 반응형 고려

### 필수 수정 포인트

* ShippingManagement 구조 재설계
* ShipmentForm 구조 재설계
* 품목-거래처 연결 관리 추가
* Product-first 출고 흐름 반영
* 기본단가 자동 반영
* 세금계산서 흐름 단순 명확화

## 10. 서버/도메인 진행 기준

이 프로그램은 **도메인을 연결하여 서버에 올리는 실제 서비스 형태**로 진행한다.

### 기본 서버 구조 원칙

* 프론트엔드와 백엔드를 분리 가능한 구조
* DB 기반 ERP 데이터 관리
* 관리자 로그인 기반 접근
* 도메인 연결 가능 구조
* 배포 후 운영 가능한 환경

### 필수 서버 고려 사항

* 사용자 인증 / 로그인
* 메뉴별 데이터 CRUD API
* 거래처 / 품목 / 재고 / 출고 / 세금계산서 / 정산 테이블 분리
* 로그 기록
* 기본 권한 체크
* 운영 DB 백업 가능 구조
* 관리자만 접근 가능한 ERP 구조

### 도메인 연결 기준

* ERP 전용 도메인 또는 서브도메인 사용 가능
* 예: erp.회사도메인
* 도메인 연결 후 SSL 적용 가능한 구조
* 관리자 로그인 페이지부터 보호된 접근 구조

### 운영 기준

* 서버 배포 후 실제 사용 가능해야 함
* 샘플 화면이 아니라 실제 DB 저장/조회/수정/삭제 가능해야 함
* 도메인 연결 이후에도 메뉴 구조와 URL 체계가 깔끔해야 함

---

## 11. 최종 원칙 정리

* 이 프로그램은 ERP다
* 쇼핑몰 기능은 절대 넣지 않는다
* 실제 운영 가능한 프로그램으로 완성한다
* 너무 무겁게 가지 않되, 핵심 기능은 완성한다
* 품목 기준 출고 흐름을 핵심 구조로 한다
* 거래처, 품목, 재고, 출고, 세금계산서, 정산이 자연스럽게 연결되어야 한다
* 개발 후 도메인 연결과 서버 운영까지 가능한 구조로 간다
* 작업자가 빠르게 이해하고 사용할 수 있어야 한다
* 기존 폴더 구조를 살리되 출고 구조는 반드시 재설계한다
* 개발을 후순위로 두지 않고 처음부터 실제 구현 기준으로 설계한다
