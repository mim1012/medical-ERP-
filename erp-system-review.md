### 1. 핵심 업무 흐름이 끊겨있음

```
정상적인 ERP 흐름:
┌─────────────────────────────────────────────────┐
│  입고/발주 → 재고 증가 → 출고/판매 → 재고 감소  │
│       ↓           ↓           ↓           ↓     │
│    발주서      재고DB    거래명세서   세금계산서  │
└─────────────────────────────────────────────────┘

현재 시스템:
┌─────────────────────────────────────────────────┐
│  입고/발주 ✅                                    │
│       ↓                                         │
│    재고 관리 ❌ (구현 안 됨)                     │
│       ↓                                         │
│  출고/판매 ❌ (구현 안 됨)                       │
│       ↓                                         │
│  세금계산서 ✅ (단독으로만 존재)                 │
└─────────────────────────────────────────────────┘

❌ 결과: 물건을 받을 수만 있고, 팔 수가 없음!
```

### 2. 의료기기 ERP의 핵심 기능 누락

#### ❌ 시리얼 번호 추적 시스템 없음
```
의료기기 = 제품 하나하나가 고유 ID(시리얼 번호)를 가짐

필수 추적 정보:
- SN-2024-00001 → 서울대병원 납품 (2024-03-01)
- SN-2024-00001 → A/S 1차 (2024-06-15)
- SN-2024-00001 → A/S 2차 (2024-09-20)
- SN-2024-00001 → 현재 위치: 서울대병원 3층 수술실

❌ 재고 관리 페이지가 없어서 이 정보를 추적 불가!
```

#### ❌ LOT 번호 추적 시스템 없음
```
소모품 = LOT 번호로 제조일자/유효기간 관리

필수 추적 정보:
- LOT-20240301 → 제조일: 2024-03-01, 유효기간: 2027-03-01
- LOT-20240301 → 입고 수량: 1,000개
- LOT-20240301 → 출고 내역: 강남병원 100개, 삼성병원 50개
- LOT-20240301 → 남은 수량: 850개

❌ 재고 관리 페이지가 없어서 유효기간 관리 불가!
❌ 리콜 발생 시 어떤 병원에 납품했는지 추적 불가!
```


### 3. 시리얼/LOT 추적 데이터 구조
```tsx
// 시리얼 번호 데이터 구조
interface SerialNumber {
  serialNumber: string;        // "SN-2024-00001"
  productId: string;           // 품목 ID
  productName: string;         // "초음파 진단기 A"
  status: 'in_stock' | 'shipped' | 'in_service';
  
  // 입고 정보
  receivedDate: string;        // "2024-03-01"
  receivedFrom: string;        // "공급업체명"
  
  // 출고 정보
  shippedDate?: string;        // "2024-03-15"
  shippedTo?: string;          // "서울대병원"
  currentLocation?: string;    // "서울대병원 3층 수술실"
  
  // A/S 이력
  serviceHistory: Array<{
    date: string;
    type: string;
    engineer: string;
    description: string;
  }>;
  
  // 유효기간
  expiryDate?: string;         // "2027-03-01"
}
```
